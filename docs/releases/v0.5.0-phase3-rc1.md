# DeskPet v0.5.0-phase3-rc1 — Release Candidate 1

**Release date**: 2026-04-23
**Phase**: 3 (Installer + packaging)
**Status**: Release candidate — awaiting installer smoke VM run (see
`docs/P3-S10-installer-smoke-runbook.md`) before promotion to stable.

---

## ⚠️ Hard requirements

- **Windows 10 22H2 / Windows 11 23H2+（64-bit）**
- **NVIDIA GPU with CUDA 12.x-capable driver**
  - 启动时 `gpu_check.rs` 通过 `nvml.dll`（随 NVIDIA 驱动发布）探测；
    没有 NVIDIA 显卡会直接弹错误对话框并退出
  - **CPU-only / AMD / Intel 集显不支持**，**不是 bug 是 Phase 3 设计取舍**
- ≥ 8 GB RAM，≥ 15 GB 空闲磁盘（installer 1.5 GB + 模型 ~9 GB → `%LocalAppData%\deskpet\models\`）
- **macOS / Linux 不支持**

详见 `docs/ARCHITECTURE.md` §"Hard requirements"。

---

## Highlights

- **单文件 installer**：NSIS + MSI 双产物，双击即用，无需手动装 Python
  / CUDA Toolkit。
- **用户数据外置**：`%AppData%\deskpet\`（config / SQLite / logs）和
  `%LocalAppData%\deskpet\models\`（模型缓存）分离；卸载默认保留用户数据。
- **Supervisor 自管 backend**：Rust 侧解析 `bundle → env → dev` 三层，
  Python 崩溃自动 respawn（60s 窗口内最多 5 次）。
- **启动 UI**：中文 splash "正在启动语音服务…"，失败时弹"重试 / 打开日志
  目录 / 退出"卡片（P3-S8）。
- **完全卸载**：SettingsPanel 危险区提供 "完全卸载" 按钮，可选同时清掉
  模型缓存（P3-S9）。
- **GPU 缺失硬门**：启动时 NVML 探测未通过即弹中文错误对话框退出（P3-S2）。

## 关键变更清单（vs v0.2.0）

### 新增

- **`backend/paths.py`**：`user_data_dir()` / `user_cache_dir()` /
  `user_models_dir()` / `ensure_user_dirs()`（基于 `platformdirs`）
- **`backend/config.py`**：`resolve_config_path()`、`seed_user_config_if_missing()`、
  空 `db_path` 自动解析到 AppData
- **`tauri-app/src-tauri/src/paths.rs`**：Rust 侧镜像同一套路径约定，
  并暴露 `open_log_dir` / `open_app_data_dir` / `purge_user_data` 命令
- **`tauri-app/src-tauri/src/user_data.rs`**：安全删除 guard
  (`MIN_PURGE_COMPONENTS = 4`) + junction-aware 删除
- **`tauri-app/src/components/StartupOverlay.tsx`**：启动 splash + 错误卡片
- **`tauri-app/src/components/SettingsPanel.tsx` 危险区**：完全卸载按钮
- **`scripts/setup_user_data.ps1`**：dev provisioner（幂等；`mklink /J`
  junction 仓库模型到 LocalAppData）
- **`scripts/build_backend.ps1`**：PyInstaller --onedir 打包
- **`scripts/perf/cold_boot.py`**：冷启基线
- **`docs/P3-S10-installer-smoke-runbook.md`**：端到端 VM runbook

### 改动

- `tauri.conf.json`：`bundle.resources` 指向 `backend/dist/deskpet-backend`；
  `targets` = `["nsis", "msi"]`
- `MemoryConfig.db_path` 默认从 `"./data/memory.db"` 改为 `""`（= AppData 自动）
- `start_backend` 不再接受 `python_path` / `backend_dir` 参数；Rust 自行解析
- SHARED_SECRET 读取加 **90s 墙钟超时** + **8100 端口冲突 precheck**（P3-S8）
- GPU 缺失 / backend 启动失败不再 silently console.warn；走 StartupOverlay 中文卡片

### 修复

- Windows GBK locale 下 structlog 写 piped stdout 抛 `OSError[Errno 22]`
  → 强制 `PYTHONIOENCODING=utf-8` + `PYTHONUNBUFFERED=1`（P3-S4）
- F5 / StrictMode 下 `start_backend` 重入抢 8100 端口 → shared_secret 幂等判据（P3-S5）
- frozen backend 找不到 `_REPO_FFMPEG` → `DESKPET_FFMPEG` env 回退到仓库 portable（P3-S5）

## 升级说明（v0.2.0 → v0.5.0-phase3-rc1）

v0.2.0 用户数据存在仓库相对路径（`./data/memory.db` 等）。升级不自动迁
移；需要保留历史的话：

```powershell
# 手动迁移（仅在 old install 还在的前提下）
mkdir $env:APPDATA\deskpet\data
copy .\data\memory.db   $env:APPDATA\deskpet\data\
copy .\data\billing.db  $env:APPDATA\deskpet\data\
copy .\config.toml      $env:APPDATA\deskpet\config.toml
```

Updater 签名密钥自 v0.2.0 起已轮换一次；自动更新到本 RC 无法工作，需
手动安装 installer。v0.5.0 stable 之后 updater 恢复可用。

## 已知限制 / Phase 4 TODO

- 首启 **GUI 模型下载器** 仍未实装：installer 打包 ~1.5 GB 不含 9 GB
  模型；dev/灰度用户需先跑 `scripts/setup_user_data.ps1` 把仓库模型
  junction 到 `%LocalAppData%`。普通用户目前只能拿带模型的完整
  installer（bundle 体积会 +9 GB，超 P3-G2 预算，故 rc1 不发）。
- **logrotate**：目前日志无滚动，长期运行会单文件膨胀。
- **macOS / Linux**：安装器工具链不同，Phase 4+ 再评估。
- **Updater**：公钥 + endpoint 已就位但 rc1 阶段暂不推自动更新；
  stable 发布时单独验证一次端到端签名校验。
- **WiX Custom Action 强删 AppData**：用户体验差，不做。
